## 세션당 상한

- /execute 1회 실행 중 자동 수집 **최대 5건**
- 5건 초과 시:
  > "자동 수집 한도 도달. /reflect에서 추가 정리 권장"
- 품질 > 양: 5건 이내에서 가장 중요한 인사이트만 저장
- "순조로운 작업"은 저장하지 않음 (noise 방지)

---

## Idea-Only 사전 검증

저장 전 content를 반드시 검증한다. 메모리는 **아이디어/원칙 수준**만 허용.

### 거부 대상 (저장 금지)

- 파일 경로 포함 (`/src/`, `./`, 절대 경로)
- 코드 스니펫 포함 (코드 블록 ` ``` `, 인라인 코드 `` ` ``)
- 라인 번호 포함 (`line 42`, `L42`, `:42`)
- 특정 변수명/함수명이 핵심인 내용

### 허용 대상

- 일반 원칙, 접근법, 개념
- 기술명, 라이브러리명 (구체적 코드가 아닌 이름 수준)
- "상황 -> 문제 -> 해결" 구조의 경험 기록

### 검증 실패 시

저장하지 않고 조용히 스킵. 사용자에게 알리지 않음.

## 중복 체크 파이프라인

### 1단계: content_hash (완전 중복)

```
SHA-256(content) 계산
-> 동일 해시 존재 -> 스킵
```

### 2단계: 시맨틱 유사도 (의미적 중복)

```
memory_search(query="{저장할 content}", limit=3)
```

| 유사도 | 판정 | 행동 |
|--------|------|------|
| >= 0.90 | 중복 | 스킵 ("중복 메모리 존재") |
| 0.80 - 0.90 | 상충 가능 | AskUserQuestion: "비슷한 기존 기록이 있습니다. 대체/병존?" |
| < 0.80 | 신규 | 새로 저장 |

## related_files 자동 첨부

인사이트 저장 시 관련 파일을 자동 첨부한다.

### 규칙

1. 인사이트 발생 시점에 수정 중이던 파일 (state.json `active_files`)
2. 최대 5개 파일
3. 파일 경로는 프로젝트 루트 상대 경로

### 용도

- /onboarding --mode project 실행 시 코드 변경과 메모리 매칭
- related_files의 파일이 변경되면 메모리가 `stale_candidate`로 전환

---

## 자동 수집 플로우 (요약)

```
이벤트 감지 (트리거 6종 중 매칭)
  -> 세션 상한 확인 (5건 미만?)
  -> content 작성
  -> Idea-Only 사전 검증 (통과?)
  -> content_hash 중복 확인 (신규?)
  -> memory_search 시맨틱 중복 확인 (< 0.90?)
  -> memory_store 저장 (+ related_files 첨부)
  -> "인사이트 저장: {유형}: {1줄 요약}" 보고
```
