# 아이디어 분석: 개발자를 위한 수학 학습 사이트

> 분석일: 2026-04-01
> 판정: **CONDITIONAL (조건부 합격)**
> 총점: 5.0/10

---

## 아이디어 요약

AI 시대에도 논리력은 중요하다는 전제 하에, 개발자가 수학을 공부하면서 동시에 학습 콘텐츠를 사이트로 만들어 배포하는 프로젝트. 파이썬 코드 예제, 그래프 시각화, 학습 대시보드, 개발 업무 연계 설명, 노트/공유 기능, 후원, 사용자 행동 추적, 피드백 수집을 기획.

## 5축 분석

### 1. 구현 가능성 — 7/10

**긍정:**
- 주 10-20시간 투입 가능 → MVP 2-3개월 내 가능
- 기술적 난이도 낮음 (마크다운 콘텐츠 + 대시보드 + 노트)
- AI 보조 콘텐츠 작성으로 속도 향상 가능
- 바이브 코딩으로 빠른 프로토타이핑

**감점:**
- 기획 기능이 과다 (대시보드 + 노트 + 공유 + 후원 + 행동추적 + 피드백 = 풀 SaaS)
- 콘텐츠 작성이 최대 병목 (수학 + 파이썬 예제 + 그래프 + 업무 연계)
- 수학 콘텐츠 정확성 검증 부담

### 2. 수익화 가능성 — 3/10

**냉정한 현실:**
- 무료 대안이 압도적: Khan Academy(무료, 한국어), 3Blue1Brown(무료, 최고 시각화), freeCodeCamp(개발자 타겟, 무료), Brilliant.org(인터랙티브, 대규모 투자)
- 수학 교육 콘텐츠에 구독료 내는 한국 사용자 극소수
- 후원 모델 불안정: 교육 콘텐츠 후원은 크리에이터 대비 매우 희박
- 법적으로 개인계좌 후원 가능하나 사업소득 종합소득세 신고 필요

### 3. 시장 수요 — 5/10

**존재하지만 제한적:**
- "개발자 수학" 검색 수요는 존재 (GitHub Into-the-Math, O'Reilly 번역서 등이 증거)
- 하지만 실제 행동(돈/시간 투자)으로 이어지는 비율 낮음
- 한국 개발자 시장만으로는 규모 작음
- AI/ML 진입 희망자에게 수요 집중되나, 이들은 이미 Coursera/Udemy 사용 중

### 4. 차별화 — 4/10

**솔직히 약함:**
- "파이썬 코드 + 그래프"는 Jupyter Notebook/Google Colab으로 이미 가능 (무료)
- "개발 업무 연계 설명"은 많은 책/블로그/강의에서 이미 다룸
- "대시보드 + 노트"는 Notion + 수학 콘텐츠로 대체 가능
- Brilliant.org 수준의 인터랙티브 경험은 1인 개발로 불가
- 유일한 차별점: "한국어 + 개발자 타겟 + 파이썬 코드 예제" 조합 — 독립 서비스를 만들 만큼 강한지 의문

### 5. 지속 가능성 — 6/10

**조건부 긍정:**
- 본인 수학 공부와 연결 → 학습 동기가 콘텐츠 생산 동력
- 수학 내용 자체는 시간이 지나도 변하지 않음 (선형대수가 내년에 바뀌진 않음)
- 사용자 기능(노트, 대시보드, 행동추적)은 지속적 유지보수 필요
- 사용자 적으면 동기 저하 → 업데이트 중단 → 방치 패턴 흔함

## 추천 MVP 범위

**핵심만 남기기 (4주 MVP):**

1. **정적 사이트 + 마크다운 콘텐츠** — 수학 주제 5-10개만 (선형대수, 확률, 미적분 기초)
2. **각 주제별 파이썬 코드 예제** — 복붙 가능한 코드 + 실행 결과 스크린샷
3. **"이거 개발에서 어디 쓰임?" 섹션** — 각 주제의 실무 연계 (핵심 차별점)
4. **진행률 체크리스트** — localStorage 기반 간단 체크 (서버 불필요)

**MVP에서 제거:**
- 노트/공유 기능 → 나중에
- 후원 → 사용자 확보 후
- 행동 추적 → Google Analytics로 충분
- 피드백 → GitHub Issues 또는 구글폼
- 사용자 인증/대시보드 → 서버 비용 절약

## 기술 스택 제안 & 예상 아키텍처

```
[콘텐츠 레이어]
  마크다운 + MDX (수학: KaTeX, 코드: Shiki)
     ↓
[프레임워크]
  Astro 5 (정적 사이트, 이미 경험 있음) 또는 Next.js (향후 확장성)
     ↓
[배포]
  Vercel 무료 티어 (정적), CloudFlare Pages 대안
     ↓
[인터랙티브 (필요 시)]
  React Islands (Astro) 또는 React 컴포넌트
  그래프: Chart.js, Plotly.js, d3.js 중 선택
     ↓
[분석]
  Google Analytics 4 (무료) 또는 Plausible ($9/월, 프라이버시)
     ↓
[향후 확장 시]
  Supabase (인증 + DB + 실시간) — 무료 티어로 시작
```

**추천: Astro 5** — 콘텐츠 중심 사이트에 최적화. 나중에 React Islands로 인터랙티브 요소 추가.

## AWS 비용 추정

| 시나리오 | 구성 | 월 비용 |
|----------|------|---------|
| **MVP (~100명)** | Vercel/CF Pages 무료 + GA4 무료 | **$0** |
| **초기 (~1,000명)** | Vercel Pro($20) + Supabase 무료 | **~$20/월** |
| **AWS Lightsail** | t3.micro 인스턴스 | **$3.5-10/월** |
| **성장 (~10,000명)** | AWS EC2 + RDS + S3 + CF | **$50-100/월** |

**결론:** 정적 사이트로 시작하면 **AWS 요금 $0**. 사용자 1,000명 이전에 AWS로 갈 이유 없음.

## 수익화 전략 제안

"수익화"보다 "학습 기록 공개"로 포지셔닝 전환을 권장:

| 전략 | 현실성 | 예상 수익 |
|------|--------|----------|
| 무료 공개 + 후원 (Buy Me a Coffee) | 높음 | 월 0-5만원 |
| 무료 기본 + 프리미엄 노트/퀴즈 | 중간 | 월 0-10만원 |
| 블로그/유튜브 연계 광고 | 중간 | 월 0-3만원 |
| 인프런/탈잉 강의 연계 | 낮음 (콘텐츠 양 필요) | 건당 수십만원 |

## 후원 법적 정리

| 항목 | 결론 |
|------|------|
| 개인계좌 후원 | 가능. 개인 크리에이터와 동일 취급 |
| 세금 | 사업소득 → 매년 5월 종합소득세 신고 |
| 통신판매업 등록 | 순수 후원은 불필요. 유료 콘텐츠 "판매" 시 등록 필요 |
| 추천 방법 | Buy Me a Coffee, Ko-fi, Toss 후원 링크 |

## 리스크 & 주의사항

1. **콘텐츠 부채:** 수학 공부 속도 < 콘텐츠 제작 속도가 되면 업데이트가 멈춤. "공부하면서 만든다"는 장점이자 가장 큰 리스크
2. **무료 경쟁:** Khan Academy, 3Blue1Brown이 이미 하고 있는 것을 1인이 이기기는 구조적으로 불가. "이기는" 게 아니라 "다른 것"을 해야 함
3. **사용자 이탈:** 수학 학습 사이트의 리텐션은 매우 낮음. "이번에는 진짜 수학 공부하겠다" → 1주일 후 안 옴
4. **기능 과다:** 기획된 기능을 전부 만들면 MVP가 아닌 풀 제품. 출시 전 지칠 위험 높음
5. **완성 자체의 가치:** 사용자 수에 집착하기보다 "직접 만들어 공개했다"는 학습 결과물 자체에 의미를 두는 편이 현실적

## 최종 판정

**CONDITIONAL (조건부 합격)** — "학습 프로젝트"로 재정의하면 진행 가치 있음. "수익화 서비스"로 접근하면 실패 확률 높음.

**조건:**
1. MVP 범위를 정적 사이트 + 5-10개 주제로 극적 축소
2. 수익화 목표를 버리고 "Learning in Public"으로 포지셔닝
3. 서버/인프라 비용 $0에서 시작 (Vercel/CF Pages)
4. 노트/공유/대시보드는 사용자 피드백 확인 후 추가
