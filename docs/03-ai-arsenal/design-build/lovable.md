# Lovable

> AI 빌더 셋 중 **유일하게 SOC 2 / ISO 27001 / GDPR 보유**. 엔터프라이즈가 끌릴 이유.

## 한줄 요약

자연어로 풀스택 React 앱을 만드는 AI 빌더. v0/Bolt 대비 **비개발자 친화도와 엔터프라이즈 컴플라이언스**가 가장 강하다. MVP 한 방에 만들고 그대로 운영 들어가기 좋음.

## 강점 (이럴 때 쓰면 좋다)

- **깨끗한 React 코드** — 셋 중 가장 정돈된 결과물. 인수인계/협업이 편함
- **풀스택 통합** — DB(Postgres), 인증, 결제(Stripe) 등을 자연어로 붙임
- **엔터프라이즈 컴플라이언스** — SOC 2 Type 2 + ISO 27001:2022 + GDPR. v0·Bolt에는 없음
- **비개발자 친화 최강** — UI/UX가 가장 평이. 코드 모르는 PM/디자이너도 작동시킴
- **빠른 MVP** — "0 → 운영 가능 앱"까지 가는 거리가 가장 짧음
- **Supabase 네이티브** — 백엔드 표준 스택과 매끄럽게 연동

## 약점 (이럴 때는 피하자)

- **프레임워크 자유도 좁음** — React 중심. Vue/Svelte 등 다른 스택 안 됨
- **UI 미감은 v0보다 약함** — production 디자인 품질은 v0가 우위
- **자유도가 Bolt보다 낮음** — 정해진 패턴 안에서 작업. "이런 방식으로 하고 싶어"가 안 먹힐 때 있음
- **가격이 약간 더 비쌈** — 무료/저가 티어가 v0·Bolt보다 짠 편

## 실전 팁

### 가장 효율적으로 쓰는 법

1. **MVP 1개를 풀스택으로 끝낸다는 마인드** — 컴포넌트 단위라면 v0가 낫다. Lovable은 앱 1개 완주에 강점
2. **Supabase 연결을 일찍** — DB/auth/storage를 초반에 붙이면 이후 작업이 매끄러움
3. **명확한 사용자 흐름 설계 후 시작** — "로그인 → 대시보드 → 결제" 같은 흐름을 먼저 정리하고 시작하면 결과물이 정돈됨
4. **GitHub 동기화 활성화** — 코드를 깃에 동기화해서 IDE에서도 손볼 수 있게

### 이 도구에서만 잘 되는 것

- **사내 도구 (Internal Tool)** — 데이터 입력 폼, 어드민 패널 같은 풀스택 사내 도구
- **컴플라이언스 필요한 MVP** — 헬스케어/금융 등 규제 환경에서 빠르게 시작
- **비개발자가 메인터너인 프로젝트** — PM/마케터가 직접 유지보수 가능

### 다른 도구로 전환해야 할 신호

- production 디자인 품질이 결정적이면 → **v0** (UI 미감)
- 다른 프레임워크 실험하고 싶으면 → **Bolt** (자유도)
- 단순 프로토타입 1개만 보면 되면 → **Claude Artifacts** 또는 **v0** (가격/속도)

## 실무 조합

- **Lovable → Claude Code** — Lovable로 풀스택 MVP 완성 후 GitHub 동기화 → Claude Code로 production화
- **v0 (컴포넌트) → Lovable (앱 조립) → Claude Code (정리)** — 대행사 권장 스택
