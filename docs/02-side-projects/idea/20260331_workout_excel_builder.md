# 아이디어 분석: 운동인을 위한 커스텀 엑셀 프로그램 빌더

> 분석일: 2026-03-31
> 판정: **HOLD**
> 총점: 4.2/10

---

## 아이디어 요약

웨이트 트레이닝 사용자를 위한 웹 기반 운동 프로그램 엑셀 빌더. 드래그앤드롭으로 운동 루틴을 구성하고, 대체 운동 토글(헬스장 상황에 맞춰 즉석 교체), 1RM 계산기, 운동 대시보드를 제공한다. 1차 MVP는 엑셀 생성 웹사이트, 2차로 모바일 앱으로 확장하여 엑셀보다 편리한 UI를 목표로 한다.

## 5축 분석

### 1. 구현 가능성 — 6/10

주당 10-15시간 투입 시, 드래그앤드롭 엑셀 빌더 + 운동 DB + 엑셀 다운로드 기능의 MVP는 2-3개월 내 가능하다. 기술적 난이도는 중간. 다만 드래그앤드롭 UI는 생각보다 UX 설계와 구현 공수가 크며, 운동 데이터베이스(운동명, 근육군, 대체운동 매핑)를 직접 구축해야 하는 부담이 있다. 엑셀 생성 라이브러리(SheetJS, ExcelJS 등)는 성숙되어 있어 기술적 리스크는 낮다.

**감점 요인:** 2차 목표인 모바일 앱까지 가면 사실상 엑셀 빌더와 완전히 다른 제품을 처음부터 만드는 것과 같다. 1차→2차 코드 재활용률이 매우 낮을 수 있다.

### 2. 수익화 가능성 — 3/10

**핵심 문제:** 사용자가 돈을 낼 이유가 매우 약하다.

- Strong 앱은 무료, Hevy는 무료 티어가 관대하며 Pro가 월 $8.99
- 운동 엑셀 템플릿은 인터넷에 무료로 넘친다 (Vertex42, Template.net, Sourcetable 등)
- "엑셀을 쉽게 만들어주는 도구"에 과금하기 어렵다 — 엑셀 자체가 무료 대안(구글 시트)이 있고, 한 번 만들면 반복 구매 동기가 없다
- 구독 모델 적용이 부자연스럽다: 엑셀 빌더는 일회성 사용에 가까움

**가능한 시나리오:** 프리미엄 운동 프로그램 템플릿 판매, 퍼스널 트레이너 대상 B2B 과금. 하지만 이는 아이디어의 본질(개인 사용자 대상)과 다른 방향이다.

### 3. 시장 수요 — 4/10

운동 기록의 불편함은 실재하는 문제이며, 피트니스 앱 시장은 2026년 $13.92B 규모로 성장 중이다. 그러나:

- **시장의 흐름은 엑셀에서 앱으로 이동 중이다.** 엑셀 빌더를 만드는 것은 트렌드에 역행한다.
- 엑셀을 고집하는 사용자는 이미 직접 만들 능력이 있거나, 기존 템플릿으로 충분한 경우가 많다.
- "엑셀 커스텀이 어렵다"는 문제를 가진 사용자는 차라리 앱(Strong, Hevy)으로 이동하는 게 자연스럽다.
- Jefit의 비교 기사("Spreadsheet vs Workout Tracker App")도 앱 우위를 강조한다.

**실제 TAM(접근 가능 시장):** 엑셀에 운동 기록을 하고 있으나 커스텀에 불편을 느끼면서도 앱으로 넘어가지 않은 사용자 → 니치 중의 니치.

### 4. 차별화 — 3/10

- **"대체 운동 토글"**은 흥미로운 기능이지만, 대부분의 워크아웃 앱(Hevy, JEFIT 등)이 이미 운동 교체 기능을 제공한다.
- Sourcetable은 이미 AI로 운동 엑셀 템플릿을 자동 생성해준다.
- 1RM 계산기와 대시보드는 거의 모든 경쟁 앱에 있는 기본 기능이다.
- "웹에서 드래그앤드롭으로 엑셀을 만든다"는 포맷 차별화이지, 기능 차별화가 아니다. 사용자 관점에서 "엑셀 파일이 나온다"는 것 자체가 이점이 되지 않는다.

### 5. 지속 가능성 — 5/10

엑셀 빌더 자체의 유지보수 부담은 낮다. 한 번 만들면 추가 개발 없이도 동작한다. 다만:

- 운동 데이터베이스(운동명, 대체운동 매핑)는 지속적 업데이트가 필요
- 사용자 참여(engagement)가 낮을 가능성이 높다 — 엑셀을 만들면 다시 방문할 이유가 약함
- 2차 목표(앱)로의 전환은 사실상 새 프로젝트 시작과 동일
- 1인 운영은 가능하지만, 성장동력이 부족

| **Hevy** | 모바일 앱 | 무료 / Pro $8.99/월 | 소셜 기능, 크로스 디바이스 싱크 |

**핵심 위협:** 사용자가 "엑셀을 쉽게 만들고 싶다"고 생각하는 순간, 이미 더 나은 대안(앱)이 존재한다는 걸 깨닫게 된다. 엑셀 빌더는 사용자를 앱으로 보내주는 디딤돌이 될 위험이 있다.

## 리스크 & 주의사항

1. **역방향 트렌드:** 시장은 스프레드시트에서 전용 앱으로 이동 중. 엑셀 빌더는 이미 구식으로 향하는 포맷에 투자하는 것
2. **낮은 전환율:** 1차(엑셀 빌더) → 2차(앱)로의 기술적 전환 비용이 높음. 코드 재활용 어려움
3. **수익화 벽:** 일회성 도구에 구독 모델 적용이 부자연스럽고, 무료 대안이 풍부
4. **"있으면 좋겠다" 수준:** "이게 없으면 불편하다"가 아닌 "있으면 좋겠다" 수준의 문제 해결
5. **좁은 TAM:** 엑셀 고집 + 커스텀 원함 + 직접 못 만듬 + 앱 안 씀 → 매우 작은 교집합

## 최종 판정

**HOLD** — 운동 기록의 불편함이라는 실재 문제에서 출발했으나, "엑셀 빌더"라는 솔루션 방향이 시장 트렌드와 역행한다. 이미 성숙한 무료 앱들이 엑셀보다 나은 경험을 제공하고 있어, 엑셀을 더 잘 만들어주는 도구의 시장성이 약하다. **2차 목표인 모바일 앱을 1차 MVP로 직접 시작하는 것이 훨씬 합리적이며**, 이 경우 차별화 포인트(대체 운동 즉석 교체 UX 등)를 더 잘 살릴 수 있다.
