## 아이디어 요약

매일 포커 카드 5장을 뽑아서, 포커 족보(원페어/투페어/풀하우스 등)의 게임적 재미와 각 카드의 타로적 해석을 조합해 "오늘의 운세"를 제공하는 가벼운 일상 서비스. 캐릭터/일러스트가 있는 커스텀 카드 디자인으로 기존 타로 앱의 진지함과 차별화. "오늘 풀하우스 나왔는데 운세는 별로래ㅋㅋ" 같은 SNS 공유가 자연스러운 구조.

## 5축 분석

### 1. 구현 가능성 — 9/10

**기술적으로 매우 단순. MVP 2-4주 가능.**

| 기능 | 난이도 | 구현 |
|------|--------|------|
| 카드 랜덤 드로우 | 극저 | 52장 배열 셔플 |
| 포커 족보 판정 | 저 | 잘 알려진 알고리즘 |
| 카드 일러스트 52장 | 중 | AI 이미지 생성(Midjourney/DALL-E) |
| 타로 해석 텍스트 | 저 | 카드별 + 족보별 사전 작성 |
| 카드 뒤집기 애니메이션 | 저 | CSS 3D transform |
| 하루 1회 제한 | 저 | localStorage + 날짜 체크 |

- 백엔드 불필요 (클라이언트 로직으로 완결)
- 배포: Vercel/CF Pages $0
- 52장 일러스트가 가장 큰 작업이지만 AI 생성으로 1-2일

### 2. 수익화 가능성 — 6/10

**입증된 거대 시장에 진입.**

- 한국 점술 시장 약 1.4조원
- 포스텔러: 860만 가입자, 연매출 100억+, 85억 투자
- 점신: 1,900만 명 사용자
- 글로벌 점성술 앱 시장: $3B → $9B (2030)
- 수익 모델: 리워드 광고(추가 뽑기) + 프리미엄(상세 해석/과거 기록)

**감점:** 대형 앱(점신/포스텔러)과 경쟁해야 하지만, "포커 운세"는 다른 포지셔닝으로 보완재 공존 가능

### 3. 시장 수요 — 7/10

**"오늘의 운세"는 검증된 일상 습관.**

- 점신 1,900만, 포스텔러 860만 = 한국 운세 수요 의심 여지 없음
- 타로가 MZ세대에서 트렌딩
- "매일 한번 뽑기" = DAU 구조 자연 형성
- 포커 족보의 도박적 쾌감 = 참여 동기 증폭

**감점:** "포커+타로" 조합의 시장 검증은 추론 기반

### 4. 차별화 — 7/10

**"포커 족보 + 타로 해석" 조합은 현재 존재하지 않음.**

| 기존 타로 앱 | 포커 타로 |
|-------------|---------|
| 진지한 분위기 | 가벼운 게임 감각 |
| 카드 해석만 | 족보 쾌감 + 해석 |
| 공유 동기 약함 | "오늘 풀하우스!" SNS 공유 강함 |
| 타로 관심자만 | 포커/게임 좋아하는 사람도 유입 |

- 트럼프 카드와 타로의 전통적 대응 관계 존재 (Hearts=Cups, Spades=Swords 등)
- 족보라는 게임적 요소가 SNS 바이럴 트리거
- 기능 복제는 쉽지만 "브랜딩+일러스트+톤"이 핵심 → 모방 장벽

### 5. 지속 가능성 — 7/10

**"매일 1회 뽑기"가 만드는 리텐션 루프.**

- 콘텐츠(카드 해석) 한번 작성 → 반영구적
- 정적 사이트 → 서버 비용 $0
- 시즌 이벤트(신년/발렌타인/크리스마스) 특별 카드 → 복귀 동기
- LLM API로 해석 다양화 가능 (매일 다른 뉘앙스)
- 향후 확장: 주간/월간 운세, 궁합 (2인 동시 뽑기), 운세 기록 달력

## 추천 MVP 범위

**"한 페이지 웹앱" (2-4주)**

1. **카드 5장 뽑기** — 뒤집기 애니메이션 + 커스텀 일러스트 카드
2. **포커 족보 판정** — "오늘은 투페어!" 연출 (시각적 하이라이트)
3. **타로 해석** — 각 카드의 의미 + 족보 기반 종합운 (연애/재물/건강)
4. **SNS 공유** — 카드 결과를 이미지로 변환 → 카카오톡/인스타 공유
5. **하루 1회 제한** — "내일 다시 오세요" (리텐션 장치)

**MVP에서 제거:**
- 사용자 계정/로그인 → 나중에
- 운세 기록/달력 → 나중에
- 프리미엄 과금 → 사용자 확보 후
- 궁합/커플 모드 → 확장 기능

## 기술 스택 제안 & 예상 아키텍처

```
[카드 일러스트]
  Midjourney / DALL-E 3 / Stable Diffusion
    → 52장 + 카드 뒷면 1장 = 53개 이미지
    → WebP 최적화 (카드당 ~50KB, 총 ~2.5MB)

[프론트엔드]
  Next.js (App Router) 또는 Astro + React Islands
    - 카드 뒤집기: CSS 3D transform + GSAP
    - 족보 판정: 클라이언트 JS
    - 해석 텍스트: JSON 데이터 (52장 × 해석)
    - 하루 1회: localStorage + date check

[SNS 공유]
  html2canvas 또는 서버사이드 OG image 생성
    → 카카오톡/트위터/인스타 공유용 이미지 자동 생성

[배포]
  Vercel ($0) — 정적 사이트 + Edge Functions(OG image)

[분석]
  Google Analytics 4 (무료)
  + 간단한 이벤트 트래킹 (뽑기 수, 공유 수)

[향후 확장]
  Supabase (사용자 계정, 운세 기록, 랭킹)
  LLM API (운세 텍스트 다양화)
```

## 수익화 전략 제안

| 단계 | 전략 | 예상 수익 |
|------|------|----------|
| **MVP (0-3개월)** | 무료. 광고 없음. 사용자 확보 집중 | $0 |
| **초기 (3-6개월)** | 리워드 광고: 광고 보면 추가 1회 뽑기 | 월 5-20만원 |
| **성장 (6개월+)** | 프리미엄: 상세 해석, 운세 기록, 광고 제거 ($2-3/월) | 월 20-100만원 |
| **확장** | 시즌 카드팩 판매, 커플 궁합 모드, B2B(브랜드 콜라보 카드) | 변동 |

**카카오톡 미니앱** 또는 **인스타 필터** 형태로 바이럴 채널 확장 고려.

## 리스크 & 주의사항

1. **일러스트 품질:** AI 생성 52장 카드의 스타일 일관성이 서비스 첫인상을 결정. 프롬프트 엔지니어링으로 통일감 확보 필수. 한 세트 다 만든 후 일관성 검토
2. **운세 해석 깊이:** "재미로 보는 것"이지만 너무 얕으면 재방문 동기 없음. 족보 × 카드 조합별로 충분한 텍스트 분량 필요
3. **법적 이슈:** 운세/점술 서비스는 한국에서 법적으로 문제없음(표현의 자유). 다만 "정확하다"는 표현 금지
4. **리텐션 증발:** 초기 호기심 후 1-2주 만에 이탈하는 전형적 패턴. "연속 뽑기 보너스"나 "주간 통계" 등 리텐션 장치 필요
5. **대형 앱 복제:** 점신/포스텔러가 "포커 운세" 기능을 추가할 수 있음. 속도로 선점하고 브랜딩으로 차별화
