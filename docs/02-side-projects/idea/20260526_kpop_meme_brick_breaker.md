# 아이디어 분석: 배속 벽돌깨기 (음악 결합)

> 분석일: 2026-05-26
> 판정: **CONDITIONAL** (AI 오리지널 음원 피벗 기준)

> **요약:** K-pop/밈 "실곡"을 쓰는 원안은 저작권 벽으로 **HOLD(4.8)**. 음악 축을 **AI 생성 오리지널 중독성 음원**으로 피벗하면 저작권 블로커가 해소되어 **CONDITIONAL(6.0)**으로 상향. 아래 [재평가 섹션](#재평가-ai-오리지널-음원-피벗)이 현재 유효 판정.

## 아이디어 요약

벽돌깨기 게임에 K-pop 인기곡·밈 음악을 배경음으로 깔고, **부서진 벽돌 수가 많을수록 곡의 재생 속도를 배속**하는 메커닉을 핵심으로 한다. 벽돌이 많이 깨질수록 음악이 빨라지면서 긴장감과 쾌감을 동시에 키워 중독성을 강화하는 구조. 모바일 게임으로 출시해 접근성을 높이고, 글로벌 시장을 타겟으로 광고(리워드) 기반 수익을 기대한다.

사용자 조건: 주 16~30시간 투입 가능 / 게임 개발 취미 수준(Unity·Godot 토이 경험) / 글로벌+광고 모델 / **저작권 전략 미정(본 분석의 핵심 검증 대상)**.

**결론 선요약:** 게임 메커닉 자체는 신선하고 만들 수 있다. 그러나 "유명 K-pop/밈 곡"을 쓰는 순간 **1인 개발자가 합법적으로 해결 불가능한 라이선싱 벽**에 부딪힌다. 이 게임은 음악이 곧 제품인데, 그 음악이 1인 개발자가 가질 수 없는 자원이다.

## 5축 분석

### 1. 구현 가능성 — 5/10

**게임 자체(긍정):** 벽돌깨기 + 물리(공 반사, 패들) + "벽돌 수 → 재생 배속" 메커닉은 Unity/Godot에서 충분히 구현 가능. 배속은 오디오 엔진의 pitch/time-stretch(예: time-stretch로 피치 유지, 또는 단순 pitch-up)로 처리 가능한 표준 기능. 주 16~30시간이면 플레이 가능한 MVP를 수개월 내 도달 가능.

**치명적 외부 의존성(감점):** 이 아이디어의 "구현"은 코드만이 아니다. **핵심 자산(인기곡)을 합법적으로 확보하는 것이 구현의 일부**인데, 이건 코드로 해결 불가능하다. 음악 게임에서 곡 라이선스는 Sync License(작곡) + Master Use License(음원) 두 개가 필요하고, 곡당 통상 $600~1,500(인기곡·영구 라이선스는 훨씬 높음). 업계 1위 Beatstar조차 UMG/Sony/Warner/BMG 등 메이저 전체와 글로벌 딜을 맺어 850+곡을 확보한 회사형 사업이며, 가이드들은 노골적으로 "indie developers don't stand a chance"라고 명시.

→ **기술 난이도는 낮으나, 제품의 본질(음악)을 합법적으로 조달하는 난이도가 1인 개발자 범위를 벗어난다.** 기술만 보면 7점이나 외부 의존성이 끌어내려 5점.

### 2. 수익화 가능성 — 3/10

- **합법성 전제 붕괴:** 무허가 곡으로는 앱스토어 수익화가 원천 차단된다. Apple/Google은 권리자 신고 시 즉시 앱을 내리며, 무허가 IP로 광고 수익을 내는 건 명백한 침해. 즉 "글로벌+광고" 모델은 **합법 음원 확보 없이는 0원**.
- **광고 모델의 구조적 한계:** 합법화하더라도 하이퍼캐주얼 광고 모델은 Voodoo/Ketchapp 같은 UA(유저 확보) 머신과 경쟁. ARPU가 극히 낮아 수백만 DAU 규모가 없으면 라이선스 비용도 못 건진다. 1인 개발자의 UA 예산으로는 비현실적.
- 즉 수익이 라이선스 비용을 초과하는 손익분기 자체가 1인 규모에서 성립하기 어렵다.

### 3. 시장 수요 — 6/10

- 벽돌깨기는 검증된 중독성 장르, K-pop은 거대한 글로벌 팬덤, 음악+캐주얼 결합 수요는 Beatstar(수천만 다운로드)가 증명. 수요 자체는 분명히 존재.
- "배속" 메커닉은 짧은 세션·즉각적 쾌감이라는 모바일 캐주얼 트렌드에 부합.
- 다만 수요가 향하는 곳은 "내가 아는 그 곡"이다. 곡이 없으면 수요도 같이 사라지는 구조라 라이선스에 종속된 수요. 그래서 만점은 못 준다.

### 4. 차별화 — 7/10

- **"벽돌 수 → 재생 배속" 메커닉은 진짜로 신선하다.** 진행도가 음악 속도로 환원되는 청각적 피드백 루프는 기존 벽돌깨기/리듬게임에서 드문 발상이고, 본 아이디어의 가장 강한 자산.
- 단, 차별화의 강도가 "유명 곡"에 크게 의존한다. 로열티프리 음원으로 바꾸면 메커닉은 남지만 "K-pop 벽돌깨기"라는 후킹은 사라져 "또 하나의 벽돌깨기"로 회귀. → 메커닉 자체는 7점이나 곡 의존성이 조건부.

### 5. 지속 가능성 — 3/10

- **콘텐츠 종속이 극심:** 음악 게임은 신곡 추가가 곧 리텐션. 신곡마다 라이선스 협상·비용·법적 노출이 반복된다. 1인이 곡 파이프라인 + 권리 협상 + 라이브옵스 + UA를 동시에 장기 운영하는 건 비현실적.
- **법적 리스크의 비대칭:** HYBE/SM/JYP는 IP 집행이 극도로 공격적(SM의 KWANGYA 119는 월 평균 400건 침해 신고 접수, HYBE/ADOR 무관용·무합의 정책). 침해 적발 시 다운 + 수익 환수 + 소송 가능성. 1인 개발자가 감당할 비대칭 리스크.
- **배속 = 회피 안 됨:** 재생 속도 변경은 2차적 변형(derivative)일 뿐 침해를 면제하지 않는다. "배속해서 쓰니까 괜찮다"는 면책 근거가 되지 못한다.

## 리스크 & 주의사항

1. **무허가 K-pop 곡 사용 = 사업 불가.** 출시 즉시 또는 적발 즉시 다운, 수익 환수, 소송 리스크. "밈이 된 노래"도 전부 저작권이 살아있다.
2. **배속 처리는 면책 사유 아님.** 변형해도 master/sync 권리는 유효.
3. **합법화 비용이 1인 규모를 초과.** 곡당 수백~수천 달러 × 다곡 × 영구 갱신. 광고 ARPU로 회수 불가.
4. 한국 메이저 레이블의 IP 집행은 세계 최고 수준으로 공격적 — 가장 쓰고 싶은 곡이 가장 위험하다.

## 최종 판정

**HOLD (4.8/10)** — 게임 메커닉(배속 루프)은 신선하고 구현 가능하나, "유명 K-pop/밈 곡"이라는 핵심 자산이 1인 개발자가 합법적으로 조달할 수 없는 자원이다. 음악이 곧 제품인 사업에서 그 음악을 못 가지면 제품이 성립하지 않는다. **아이디어를 버릴 게 아니라, 음악 축을 피벗하면 살릴 수 있다.**

### 피벗 제안 (살리는 방향)

1. **메커닉 중심 + 라이선스프리 음원** — "배속 벽돌깨기"의 신선한 메커닉을 핵심 IP로 삼고, 로열티프리/저작권프리(예: NCS, Epidemic Sound 라이선스, AI 생성 트랙) 음원으로 출시. 후킹은 약해지나 합법·지속가능. **가장 현실적.**
2. **BYO(사용자 음원 업로드)** — 사용자가 자기 음악 파일을 넣어 플레이. 저작권 책임을 사용자에게 이전(앱 자체는 곡을 배포하지 않음). 비트 분석으로 벽돌 배치 자동 생성하면 차별화 + 무한 콘텐츠. (단 스토어 정책·기기 파일 접근 검토 필요)
3. **신인/인디 아티스트 프로모 딜** — 노출을 원하는 라이징 아티스트와 무상/저가 프로모션 라이선스. 게임이 곡 홍보 채널이 되는 윈윈. 1인이 협상 가능한 규모.

→ 이 중 하나로 음악 조달을 해결하면 **재분석 시 CONDITIONAL~PASS 가능**. 특히 (1)+(2) 결합("내 음악으로 즐기는 배속 벽돌깨기")이 1인 개발자에게 가장 합리적.

### 판정: CONDITIONAL (6.0/10)

평균 6.0 (≥5), 최저 5 (≥3) → **CONDITIONAL** 성립. (PASS는 평균 7+ 필요, 미달.)
저작권 블로커가 사라진 대신, **남은 관건은 "마케팅 훅 없이 게임플레이만으로 사용자를 끌 수 있는가"와 "1인의 하이퍼캐주얼 수익화"**로 이동.

### 진행 조건 (CONDITIONAL의 "조건")

1. **메커닉 펀(fun) 검증 우선.** "K-pop 곡"이라는 목발이 없어진 만큼, 배속 루프 자체가 재밌는지 **그레이박스 프로토타입으로 먼저 검증**. 이게 안 되면 나머지는 무의미.
2. **AI 음악 큐레이션 품질.** Suno 곡을 그냥 깔면 "AI 양산 BGM" 느낌. 배속 시 더 신나게 들리는 곡을 선별·후처리하는 안목/파이프라인이 차별화의 핵심.
3. **수익화 현실화.** 광고 단독 대신 광고+IAP(스킨/광고제거/곡팩) 하이브리드로 ARPU 보강. UA 예산 없이 오가닉(틱톡/쇼츠 바이럴) 가능한 후킹 영상 만들 수 있는지 사전 점검.

### 추천 MVP 범위 (피벗 기준)

1. 벽돌깨기 코어 (공 물리, 패들, 벽돌 그리드) — 1개 스테이지
2. **배속 메커닉**: 부순 벽돌 수 누적 → 음악 재생 배속 단계 상승 (피치 유지 time-stretch 또는 단계별 pitch-up)
3. AI 생성 오리지널 트랙 3~5곡 + 곡별 벽돌 레이아웃
4. 점수/콤보 + 리트라이 루프 (중독성 핵심)
5. 리워드 광고 1개 지점 (부활/2배 보상)

### 기술 스택 제안

- **엔진**: Unity (모바일 캐주얼 표준, 에셋스토어·광고 SDK 생태계) 또는 Godot(무료·경량). 취미 경험 고려 시 자료 많은 **Unity** 권장
- **오디오**: Unity Audio + time-stretch 플러그인(또는 FMOD)으로 피치 유지 배속. 비트 동기화는 곡별 BPM 메타데이터로 사전 처리
- **음원**: Suno/Udio Pro 구독으로 오리지널 트랙 생성·큐레이션
- **수익화 SDK**: AdMob 또는 LevelPlay(구 ironSource) 미디에이션
- **아키텍처**: 단일 앱, 곡+레이아웃을 데이터(JSON/ScriptableObject)로 분리해 콘텐츠 추가를 코드 변경 없이 처리

### 수익화 전략 제안

- **하이브리드**: 리워드 광고(부활/보상 2배) + 인터스티셜(스테이지 간) + IAP(광고 제거, 곡팩, 패들/공 스킨)
- **오가닉 UA 우선**: "음악이 미친듯이 빨라지는 순간" 클립을 틱톡/쇼츠/릴스로 — 배속 메커닉은 본질적으로 짧은 영상에 잘 맞음. 유료 UA 없이 바이럴 시도
- 라이선스 비용 0이므로 손익분기가 원안보다 압도적으로 유리

### 남은 리스크

1. **AI 양산 BGM의 평범함.** 큐레이션 실패 시 "그저그런 무료 BGM 벽돌깨기"로 전락. 음악 안목이 차별화의 생사를 가름.
2. **하이퍼캐주얼 레드오션.** 벽돌깨기는 포화 장르. 배속 메커닉 + 음악 시너지가 충분히 신선해야 묻히지 않음.
3. **오가닉 바이럴 실패 시 무명.** UA 예산 없는 1인은 발견(discoverability)이 최대 적. 후킹 영상 제작 역량이 사실상 필수 스킬.
4. AI 음악 서비스 약관·법적 환경은 유동적 — Pro 플랜 상업권 조항을 출시 시점에 재확인 필요.
