주어진 주제에 대해 사용자의 수준에 맞는 교육 자료를 생성한다. 핵심 내용을 파악하고, 사용자의 경험/지식 수준을 문답으로 조사한 뒤, 맞춤형 학습 문서를 작성한다.

## 파라미터

- `$ARGUMENTS` (필수): 학습 주제 (자연어, spec 경로, 기술명 등)
- `--level` (선택): 수준 직접 지정 (입문/초급/중급/고급). 지정 시 Step 2 레벨 조사 스킵

---

## Step 1: 핵심 내용 파악

주제를 분석하여 학습 범위를 정의한다.

**주제가 프로젝트 코드인 경우 (serena):**
- `mcp__serena__get_symbols_overview(relative_path="{파일}")` → 코드 구조 파악
- `mcp__serena__find_symbol(name_path="{클래스/함수}", depth=1)` → 핵심 클래스/함수 목록
- `mcp__serena__find_referencing_symbols(symbol_name="{심볼}")` → 사용 패턴 확인
- 파일 전체 Read 대신 심볼 단위 분석으로 토큰 절약
- **fallback:** serena MCP 미연결 시 기존 Read + Grep으로 대체

**주제가 spec 경로인 경우:**
- spec 문서를 읽고 사용된 기술/개념/패턴 추출
- 관련 코드가 있으면 구조 파악

**주제가 기술명/개념인 경우:**
- 메모리 검색:
  ```
  CC memory(MEMORY.md) 확인(query="{주제}", limit=10)
  CC memory 확인(query="{주제}", limit=5)
  ```
- Context7 문서 조회 (라이브러리인 경우):
  ```
  mcp__context7__resolve-library-id(library="{주제}")
  → mcp__context7__query-docs(library_id="{id}", query="overview getting started")
  ```
- **WebSearch로 최신 정보 적극 수집:**
  ```
  WebSearch(query="{주제} tutorial best practices 2025 2026")
  WebSearch(query="{주제} vs {대안} comparison")
  ```
  → context7 문서와 함께 최신 블로그, 튜토리얼, 비교 자료를 교육 자료에 반영

**복잡한 주제의 학습 구조화 (sequential-thinking):**
```
mcp__sequential-thinking__sequentialthinking(thought="주제: {주제}. 학습자가 이해해야 할 핵심 개념을 선행 지식 → 기본 → 심화 순서로 구조화")
```
→ 교육 자료의 목차와 학습 순서를 체계적으로 설계
- **fallback:** sequential-thinking MCP 미연결 시 직접 구조화

**산출물:** 주제의 핵심 개념 목록 (5-15개 항목)

---

## Step 2: 사용자 수준 조사 (문답)

AskUserQuestion으로 3-5개 질문을 통해 수준을 파악한다.

**질문 설계 원칙:**
- 주제의 핵심 개념별로 "알고 있는가 / 써본 적 있는가 / 설명할 수 있는가" 판별
- 예/아니오가 아닌, 구체적 경험을 물어봄

**질문 예시 (주제: 그래프 DB):**
```
이 주제에 대해 몇 가지 여쭤볼게요:

1. 그래프 데이터베이스를 사용해본 적 있나요? (그래프 DB, ArangoDB 등)
2. Cypher 쿼리를 직접 작성해본 경험이 있나요?
3. RDB와 그래프 DB의 차이를 설명할 수 있나요?
4. 현재 이 주제를 학습하는 목적이 뭔가요? (업무 적용 / 개념 이해 / 면접 준비 등)
```

**수준 판정:**

| 수준 | 기준 | 문서 스타일 |
|------|------|-----------|
| **입문** | 핵심 개념 대부분 모름 | 비유 + 그림 + 단계적 설명 |
| **초급** | 개념은 알지만 실무 경험 없음 | 예제 중심 + 실습 가이드 |
| **중급** | 기본 사용 가능, 심화 부족 | 내부 동작 + 패턴 + 주의사항 |
| **고급** | 실무 경험 풍부, 특정 영역 심화 | 아키텍처 + 트레이드오프 + 엣지케이스 |

---

## Step 3: 문서 작성

수준에 맞는 교육 자료를 생성한다.

**저장 경로:** `docs/05-study/YYYYMMDD_{name}_study.md`

**문서 구조:**

```markdown
# {주제} 학습 가이드

> 대상: {판정된 수준}
> 생성일: {날짜}
> 학습 목적: {Step 2에서 파악한 목적}

---

## 개요
{주제가 뭔지, 왜 알아야 하는지 — 수준에 맞는 톤}

## 핵심 개념
{수준별 깊이 조절}

### 1. {개념 1}
{설명 + 예시}

### 2. {개념 2}
{설명 + 예시}

...

## 실전 적용
{현재 프로젝트에서 어떻게 사용되는지 — 해당 시}

## 주의사항
{메모리에서 발견된 gotcha/insight 포함}

## 더 알아보기
{다음 단계 학습 추천 — 수준 한 단계 위 내용}
```

**수준별 차이:**

| | 입문 | 초급 | 중급 | 고급 |
|---|---|---|---|---|
| 비유/그림 | 필수 | 선택 | 불필요 | 불필요 |
| 코드 예시 | 최소 | 풍부 | 핵심만 | 고급 패턴 |
| 내부 동작 | 생략 | 간략 | 상세 | 심화 |
| 트레이드오프 | 생략 | 언급 | 분석 | 심화 비교 |
| gotcha/주의 | 생략 | 기본 | 상세 | 엣지케이스 |

---

## Step 4: 검토 및 완료

AskUserQuestion으로 문서를 사용자에게 제시:

```
학습 자료를 작성했습니다: {경로}

확인해주세요:
  1. 수준이 적절한가요? (너무 쉽거나 어렵지 않은지)
  2. 빠진 내용이 있나요?
  3. 특히 더 자세히 알고 싶은 부분이 있나요?
```

피드백에 따라:
- 수준 조절 → Step 3 재작성
- 내용 추가 → 해당 섹션 보완
- 만족 → 완료

**완료 시:**
- 학습 이력을 메모리에 저장 (선택):
  ```
  CC memory 파일 저장(
      memory_type="preference",
      title="{사용자ID}의 {주제} 학습 수준: {수준}",
      content="{주제에 대해 {수준} 수준. {핵심 파악 내용 요약}",
      파일 기반 로컬 저장
  )
  ```

## 실패/fallback

- **MCP 미연결:** Context7/메모리 검색 없이 일반 지식 기반으로 작성
- **주제 범위 과다:** AskUserQuestion으로 범위 축소 ("어떤 부분에 집중할까요?")
- **수준 판정 불가:** 중급 기본값으로 작성, Step 4에서 조절
