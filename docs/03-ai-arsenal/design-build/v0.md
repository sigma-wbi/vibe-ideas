# v0 (Vercel)

> AI 빌더 중 **UI 미감 최강**. production-grade React 코드를 그대로 뽑아낸다.

## 한줄 요약

Vercel이 만든 AI 앱 빌더. 자연어 프롬프트로 **Next.js + shadcn/ui + Tailwind CSS** 기반의 production 수준 React 컴포넌트/페이지를 생성한다. 디자인 시안 → 코드 변환의 표준.

## 강점 (이럴 때 쓰면 좋다)

- **UI 품질 최상** — Lovable·Bolt와 비교해도 미감과 마무리 디테일이 가장 좋음. shadcn/ui 패턴이 완성도 높게 들어감
- **production-grade 코드** — 그대로 자기 프로젝트에 복붙해도 큰 손질 없이 돌아감. Next.js App Router, React Server Components 등 최신 패턴 준수
- **Vercel 생태계 통합** — 한 번 클릭으로 Vercel 배포. Next.js / Tailwind / shadcn 스택이 표준이면 마찰 거의 없음
- **컴포넌트 단위 작업** — 앱 전체보다 "랜딩 페이지", "settings 폼", "대시보드 카드" 같은 컴포넌트 단위에서 강력
- **MCP 지원** — Cursor/Claude Code 같은 IDE가 v0 디자인을 MCP로 끌어올 수 있음

## 약점 (이럴 때는 피하자)

- **프론트엔드 전용** — 백엔드/DB/인증 등은 직접 붙여야 함. Lovable·Bolt와 달리 풀스택 X
- **프레임워크 선택권 좁음** — Next.js + React 중심. Vue·Svelte 등 다른 스택은 지원 약함
- **shadcn/Tailwind 의존** — 다른 디자인 시스템을 쓰는 회사라면 결과물을 많이 손질해야 함
- **유료 한계 빠름** — 무료 티어가 한정적이라 본격 사용하면 빠르게 유료 필요

## 실전 팁

### 가장 효율적으로 쓰는 법

1. **컴포넌트 단위로 요청** — "전체 SaaS" 보다 "프라이싱 페이지", "설정 모달" 같은 단위가 결과 품질 압도적으로 좋음
2. **shadcn 컴포넌트명 직접 지정** — "DropdownMenu와 Dialog를 써서" 같이 명시하면 더 깔끔
3. **반복 수정으로 다듬기** — 한 번에 완성 시도 X. 첫 안 → "이 부분 좀 더 미니멀하게" 식 반복
4. **코드 복사 → 자기 프로젝트** — v0 안에 영구 보관 의존하지 말고 깃 리포로 옮기기

### 이 도구에서만 잘 되는 것

- **랜딩페이지 / 마케팅 사이트** — 미감과 production 품질의 균형 최고
- **shadcn 표준 UI** — 학습 데이터 편향 덕분에 표준에서 안 벗어남
- **빠른 디자인 시안** — Figma 시안 만들기보다 빠르게 "코드로 된 시안" 생성

### 다른 도구로 전환해야 할 신호

- 백엔드/DB가 필요해지면 → **Lovable** 또는 **Bolt**
- Next.js 아닌 스택(Vue/Svelte/Astro)이 필요하면 → **Bolt** (프레임워크 자유도 1위)
- 비개발자가 직접 만들고 운영해야 하면 → **Lovable** (가장 친화적)

## 실무 조합

- **v0 → Claude Code** — v0로 컴포넌트 생성 → Claude Code로 자기 프로젝트에 통합 + 다듬기 (가장 흔한 패턴)
- **v0 → Lovable** — v0로 컴포넌트 만들고 → Lovable에서 풀스택 앱으로 조립
